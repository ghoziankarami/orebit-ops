# OpenClaw Workspace Legacy Reference

This repo is now `orebit-ops`, focused on current operations rather than the older bootstrap/dev workspace model.

## What changed

The old workspace-era structure included experimental or no-longer-canonical areas such as:

- `infra-template/`
- `rag-public/`
- `research-data/`
- Docker-first RAG bootstrap flows
- `/workspace/`-based runtime assumptions

Those are no longer treated as active operational surfaces in this repo.

## Current rule

Use these active docs instead:

- `docs/operations/OPERATIONAL_STATUS.md`
- `docs/operations/OPEN_GAPS.md`
- `docs/workflows/SECOND_BRAIN_CAPTURE_WORKFLOW.md`
- `docs/workflows/PRODUCT_DIGITAL_BLUEPRINT.md`
- `docs/setup/RCLONE_SETUP.md`

## Legacy repo reference

If older implementation history or workspace-era context is needed, refer to the legacy OpenClaw workspace repository/history rather than reintroducing that structure into `orebit-ops`.

This file exists only to preserve the migration context.
