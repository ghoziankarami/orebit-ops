POST-DEPLOYMENT VERIFICATION
================================================================================

After deployment, verify:

□ Nginx running: sudo systemctl status nginx
□ SSL active: sudo certbot certificates
□ Health check works: curl https://rag.orebit.id/health
□ API query works: curl https://rag.orebit.id/api/rag/health
□ VPS memory < 500MB: free -h
□ No ChromaDB on VPS: ps aux | grep chromadb (empty)
□ No rclone on VPS: ps aux | grep rclone (empty)
□ Can access via browser: https://rag.orebit.id

Health check should return:
{
  "status": "healthy",
  "service": "rag-api-wrapper",
  "corpus": {
    "indexed_papers": 350
  }
}

Rollback to old architecture:
□ Stop deploy-vps-frontend.sh services
□ Restore old configuration
□ Revert commit if needed

================================================================================
TROUBLESHOOTING GUIDE
================================================================================

Issue 1: Health check not accessible
Solution:
  • Check Nginx status: sudo systemctl status nginx
  • Check SSL certs: sudo certbot certificates
  • Test Nginx config: sudo nginx -t
  • Check logs: sudo tail -f /var/log/nginx/error.log

Issue 2: SSL certificate error
Solution:
  • Renew: sudo certbot renew
  • Force renew: sudo certbot renew --force-renewal
  • Re-run: sudo certbot --nginx -d rag.orebit.id

Issue 3: Can't connect to QwenPaw
Solution:
  • Test ngrok URL: curl https://ngrok-url.ngrok-free.app/health
  • Test direct IP: curl http://QWENPAW_IP:3004/health
  • Check firewall: sudo ufw status
  • Check API wrapper running on QwenPaw

Issue 4: Nginx returns 502 Bad Gateway
Solution:
  • Check proxy target accessible
  • Check QwenPaw API running
  • Verify ngrok tunnel active
  • Increase proxy timeouts in Nginx config
