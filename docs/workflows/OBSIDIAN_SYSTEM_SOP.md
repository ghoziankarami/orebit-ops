# Obsidian System SOP

This is the single best document to read if you want to understand how the current Orebit Obsidian system is supposed to work.

## What this system is for

The system is designed to turn QwenPaw conversations, research work, geology and exploration questions, mining and offshore workflows, and product-digital output into durable readable notes.

In short:
- QwenPaw is the exploration surface
- Obsidian is the durable artifact surface
- the repo is the canonical documentation and reset surface

## Core principle

Do not treat raw chat history as the knowledge base.
Treat chat as a place to think.
Treat Obsidian as the place where refined outputs live.

## The 5-layer model

### `0. Inbox/`
Use this for fast capture.
This is where unfinished, low-friction, or newly-created material should land first.

Typical lanes:
- `0. Inbox/Research/`
- `0. Inbox/Automation Inbox/`
- `0. Inbox/Papers/`
- `0. Inbox/Reading Inbox/`
- `0. Inbox/Links/`
- `0. Inbox/Task Notes/`

Rule:
- capture fast
- do not let inbox become permanent storage

### `1. Projects/`
Use this for active work with a finish line.

Important lanes:
- `1. Projects/Product Studio/`
- `1. Projects/Research Programs/`

A good project folder usually contains:
- one overview note
- core working notes
- drafts or outputs tied to the project goal

### `2. Areas/`
Use this for ongoing responsibilities.
These are things you keep managing over time.

Examples:
- Knowledge Management
- Research Practice
- Product Marketing
- Content Operations
- Sales Systems
- Ad Operations

### `3. Resources/`
Use this for durable reusable knowledge.
This is where notes should end up when they are no longer just project scratch.

Important lanes:
- `Research Notes`
- `Literature Notes`
- `Evergreen`
- `Frameworks`
- `Geology`
- `Exploration Methods`
- `Mining Systems`
- `Offshore Operations`
- `SOPs`
- `Visual Concepts`
- `Operating Systems`
- `Product Knowledge`

### `4. Archive/`
Use this for inactive, replaced, or historical material.
Archive instead of deleting useful history.

## How QwenPaw sessions should flow

### Stage 1: ask freely
Use QwenPaw for exploration, comparison, questioning, synthesis, drafting, and ideation.

### Stage 2: decide if the output matters
If the result has future value, do not leave it only in chat.
Capture it.

### Stage 3: choose the note type
Use one of these typed outputs:
- Research Brief
- Field/Domain Question
- Literature Synthesis
- Evergreen Note
- SOP Draft
- Workflow Draft
- Image Concept
- Decision Log
- Project Overview
- Research Package Overview

### Stage 4: place it in the right lane
Examples:
- raw geology question -> `0. Inbox/Research/`
- refined geology synthesis -> `3. Resources/Research Notes/` or `3. Resources/Geology/`
- reusable exploration method -> `3. Resources/Exploration Methods/`
- mining workflow note -> `3. Resources/Mining Systems/`
- offshore vessel workflow note -> `3. Resources/Offshore Operations/`
- SOP -> `3. Resources/SOPs/`
- image prompt or diagram concept -> `3. Resources/Visual Concepts/`
- project-specific deliverable -> `1. Projects/...`

### Stage 5: promote or archive
Do not keep everything forever in `Inbox`, `Fleeting Notes`, or transitional lanes.
Promote good notes.
Archive stale notes.

## How to handle research

### New research
For fresh research work:
- capture the question in `0. Inbox/Research/`
- if it becomes active work, create or place it under `1. Projects/Research Programs/`
- when mature, promote summaries and methods to `3. Resources/...`

### Legacy research
For older research like `PINN-Geostat-Augmented`:
- do not copy the entire runtime tree into Obsidian
- do not flatten everything into one giant note
- represent it as a compact research package

A good compact research package contains:
- overview
- research summary
- methods and findings
- workflow or manuscript lessons
- source lineage
- optional promotion tracker

## Simple operating rule

If a chat result matters later, turn it into a note.
If a note becomes reusable, promote it.
If a note stops being active, archive it.

## The easiest way to think about the system

- ask in QwenPaw
- capture in Inbox
- shape in Projects
- preserve in Resources
- retire in Archive
