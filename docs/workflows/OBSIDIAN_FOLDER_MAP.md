# Obsidian Folder Map

This document provides a practical folder-by-folder interpretation of the current Obsidian system.

## `0. Inbox/`

Use for quick capture, not permanent storage.

Examples:
- `Research/` for raw research questions
- `AI Sessions/` for useful but still-rough AI outputs
- `Papers/` for paper intake
- `Reading Inbox/` for reading backlog
- `Links/` and `Link Inbox/` for URLs
- `Task Notes/` for task-oriented intake

## `1. Projects/`

Use for active work that has a finish line.

Examples:
- `Product Studio/` for digital-product builds
- `Research Programs/` for active or legacy-significant research packages

## `2. Areas/`

Use for ongoing stewardship.

Examples:
- `Knowledge Management/`
- `Research Practice/`
- `Audience Research/`
- `Product Marketing/`
- `Content Operations/`
- `Sales Systems/`
- `Ad Operations/`

## `3. Resources/`

Use for durable reusable knowledge.

Examples:
- `Research Notes/` for promoted research synthesis
- `Literature Notes/` for paper syntheses
- `Frameworks/` for reusable mental models and methods
- `Exploration Methods/` for reusable exploration or geoscience method notes
- `Mining Systems/` for mining workflow/system knowledge
- `Offshore Operations/` for marine and vessel workflow knowledge
- `SOPs/` for reusable procedure notes
- `Visual Concepts/` for image or diagram prompt notes
- `Operating Systems/` for repeatable operating patterns

## `4. Archive/`

Use for inactive, replaced, or legacy material.

## Rule

When in doubt:
- capture in inbox
- promote to project or resource
- archive instead of leaving clutter in transitional lanes forever
