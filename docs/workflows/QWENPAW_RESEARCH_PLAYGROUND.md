# QwenPaw Research Playground

This document defines how QwenPaw should be used as a research playground while keeping Obsidian as the durable source of truth.

## Goal

Use chat freely for exploration, but preserve only refined outputs in the vault.

## Core principle

Do not treat raw chat history as the knowledge base.
Treat chat as a thinking surface and Obsidian as the durable artifact surface.

## Input types supported

- geology questions
- exploration-method questions
- mining operations questions
- offshore workflow questions
- SOP requests
- workflow drafts
- product-digital research
- image and diagram concept requests

## Capture flow

### Stage 1: free exploration in chat
Use QwenPaw to think, compare, question, synthesize, and draft.

### Stage 2: capture candidate output
When something is useful, first save it into a low-conflict capture lane such as:

- `0. Inbox/Research/`
- `0. Inbox/AI Sessions/`
- `0. Inbox/Papers/`

### Stage 3: classify the output
Decide what kind of value the session produced:

- research brief
- field/domain question
- literature synthesis
- evergreen insight
- SOP draft
- workflow draft
- image concept
- decision log
- project-specific asset draft

### Stage 4: promote by value type
Promote outputs according to what they are:

- research brief -> `3. Resources/Research Notes/`
- field/domain question -> `0. Inbox/Research/` first, then promote as needed
- literature synthesis -> `3. Resources/Literature Notes/`
- reusable concept -> `3. Resources/Evergreen/`
- reusable method -> `3. Resources/Frameworks/`
- mining systems knowledge -> `3. Resources/Mining Systems/`
- offshore knowledge -> `3. Resources/Offshore Operations/`
- exploration method knowledge -> `3. Resources/Exploration Methods/`
- SOP -> `3. Resources/SOPs/`
- image concept -> `3. Resources/Visual Concepts/`
- project-specific work -> `1. Projects/...`

## Suggested session outputs by request type

- geology or exploration question -> `Field/Domain Question` or `Research Brief`
- mining or offshore operations request -> `Workflow Draft`, `SOP Draft`, or domain note
- paper-reading request -> `Literature Synthesis`
- image-generation request -> `Image Concept`
- planning request -> `Decision Log` or `Workflow Draft`

## Value rule

Every important session should leave behind at least one readable note if it contains durable value.
Do not keep everything as transcript.
Keep outputs as curated notes.

## Recommended note types

- Research Brief
- Field/Domain Question
- Literature Synthesis
- Evergreen Note
- SOP Draft
- Workflow Draft
- Image Concept
- Decision Log

## Why this matters

This pattern allows QwenPaw to become a true research playground without turning the vault into a junk drawer.
