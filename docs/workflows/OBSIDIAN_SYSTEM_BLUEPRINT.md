# Obsidian System Blueprint

This document defines the intended end-state structure for the Orebit Obsidian system.

## Purpose

The vault should function as:
- a second brain
- a durable research system
- a geology, exploration, mining, and offshore knowledge base
- a product-digital operating system
- a place where AI-assisted work becomes readable artifacts

## Core design rules

- capture fast, promote slowly
- keep raw intake separate from durable knowledge
- keep active projects separate from reusable resources
- prefer small typed notes over giant mixed-purpose notes
- preserve lineage without dumping runtime clutter into the vault

## Top-level structure

### `0. Inbox/`
Fast intake and low-friction capture.

Expected lanes:
- `0. Inbox/Research/`
- `0. Inbox/AI Sessions/`
- `0. Inbox/Papers/`
- `0. Inbox/Reading Inbox/`
- `0. Inbox/Links/`
- `0. Inbox/Task Notes/`
- `0. Inbox/Journal/`

### `1. Projects/`
Active work with a finish line.

Expected lanes:
- `1. Projects/Product Studio/`
- `1. Projects/Research Programs/`
- project-specific folders

### `2. Areas/`
Long-running responsibilities and stewardship.

Expected lanes include:
- `Knowledge Management`
- `Research Practice`
- `Audience Research`
- `Product Marketing`
- `Content Operations`
- `Sales Systems`
- `Ad Operations`
- domain stewardship areas

### `3. Resources/`
Durable reusable knowledge.

Expected lanes include:
- `Research Notes`
- `Literature Notes`
- `Evergreen`
- `Frameworks`
- `Geology`
- `Exploration Methods`
- `Mining Systems`
- `Offshore Operations`
- `SOPs`
- `Visual Concepts`
- `Operating Systems`
- `Product Knowledge`
- `Markets`
- `Personas`
- `Offers`
- `Copywriting`
- `Ads`
- `Landing Pages`
- `Prompt Packs`

### `4. Archive/`
Inactive, legacy, or historical material.

## Typed note system

Use typed notes so chat output becomes structured knowledge.

Recommended note types:
- Research Brief
- Field/Domain Question
- Literature Synthesis
- Evergreen Note
- SOP Draft
- Workflow Draft
- Image Concept
- Decision Log
- Project Overview
- Research Package Overview

## Capture and promotion model

### Capture
Put first-pass material into inbox surfaces.

### Classify
Decide what kind of note it is.

### Promote
Move or rewrite into the correct durable lane.

### Archive
If no longer useful, archive rather than letting transitional lanes grow forever.

## Legacy research rule

Legacy research such as `PINN-Geostat-Augmented` should be represented as compact research packages:
- overview
- summary
- methods/findings
- workflow lessons
- source lineage
- optional promotion tracker

## AI workflow rule

QwenPaw is the exploration surface.
Obsidian is the durable artifact surface.
Do not treat raw chat transcript as the final knowledge base.
