# SOP - Automation Inbox Review

## Purpose

Review staged automation output without letting the inbox fill up with low-value transcript debris.

## Inputs

Use these review surfaces:
- `0. Inbox/Automation Inbox/README.md`
- `0. Inbox/Automation Inbox/Automation Review Queue.md`
- `0. Inbox/Review Queue.md`

## Review procedure

1. Open the automation review queue.
2. Read the candidate note once without promoting it.
3. Decide whether it is:
   - reusable
   - specific
   - worth finding again in one month
4. Choose one action:
   - promote
   - keep staged for rewrite
   - archive
   - discard

## Promotion rule

Promote only when the note is cleaner than the transcript and has a clear destination.

## Destination examples

- reusable workflow -> `3. Resources/Operating Systems/`
- reusable SOP -> `3. Resources/SOPs/`
- research synthesis -> `3. Resources/Research Notes/`
- project-specific material -> `1. Projects/...`
- still-raw material -> keep in inbox until clearer

## Rule

Low-confidence automation must stage first.
Do not let the system autopromote transcript fragments into durable knowledge.
