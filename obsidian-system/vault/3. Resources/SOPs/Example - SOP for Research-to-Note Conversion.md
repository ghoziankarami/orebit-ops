# Example - SOP for Research-to-Note Conversion

## Purpose

Convert a useful research session into a durable note without storing unnecessary transcript clutter.

## Scope

Use for research, geology, exploration, mining, offshore, or product-digital sessions.

## Preconditions

A useful conclusion, method, question, or reusable insight has emerged from a session.

## Procedure
1. identify the most reusable output
2. choose the correct note type
3. capture a first version in the appropriate inbox lane if needed
4. rewrite into a clean typed note
5. place it in the correct project or resource folder
6. add source lineage if future traceability matters

## Risks / Failure Points
- keeping raw chat instead of a refined note
- placing the note in the wrong lane
- mixing project-specific context with reusable knowledge

## Review Notes

If the note remains useful after the immediate session, promote it out of the inbox.
